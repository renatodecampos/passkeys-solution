# Project overview — passkeys-solution

## Purpose

**passkeys-solution** is a proof of concept for **passwordless WebAuthn/FIDO2 (passkey)** authentication: a **Fastify/TypeScript** backend and an **Expo/React Native** Android client, plus a **file-based agent harness** (RFCs, `tasks/`, `AGENTS.md`) used to orchestrate development with AI tools.

## Repository type

| | |
|---|---|
| **Structure** | **Monorepo** with two distinct parts |
| **Parts** | `passkeys-server/` — API backend · `passkeys-app/` — mobile client |

## Technology summary

| Part | Role | Core stack |
|------|------|------------|
| **server** | WebAuthn relying party, session, challenges | Node 20+, TypeScript, Fastify 5, MongoDB, Redis, `@simplewebauthn/server` |
| **app** | Registration/authentication UI and native passkey | Expo SDK 53, React 19, React Native 0.79, Expo Router 5, `react-native-passkey` |

## Architecture classification

- **Server:** API-centric layered layout — routes in `infra/api/`, business logic in `registration/` and `authentication/`, data in `infra/database/`, config in `setup/`.
- **App:** File-based routes under `app/`, **single HTTP surface** in `services/api.ts` (no ad-hoc `fetch` in screens for server calls).
